# Voyager Observatory — Recorded Architecture Decisions (Q1–Q10)

*Answers to the ten open questions in `REVIEW_HANDOFF.md`, produced by an Opus deep-trace agent on the `phase0-safety` branch (July 2026) and adopted as working decisions. Where a decision amends the original roadmap, that is called out explicitly.*

| # | Question | Decision | Confidence |
|---|----------|----------|------------|
| Q1 | Which parser is the future? | **voyager-ast wins.** Primary for its supported languages; regex skeletonizer as long-tail fallback; **delete `core/syntax/adapter.rs`**. The LSP PoC (`experiments/lsp_poc/comparison_results.csv`) already settled it: regex/tree-sitter ≈1.0 precision at 50–75× rust-analyzer's speed; the recall gap is mostly locals a context tool doesn't need. Only 5 of 26 LanguageId variants have real adapters — frame honestly as "excellent top ~5–12, graceful fallback elsewhere." Risk: tree-sitter C grammars vs the wasm32 target. | High |
| Q2 | Lenses and intents: one concept or two? | **Two — do NOT merge (AMENDS roadmap 2.4).** A lens modifies serialized file content; an intent produces a terminal report (relevance %, read/skim/skip, no content) — `--explore` returns before `EncoderConfig` is even built. MCP already exposes them as separate tools with different JSON shapes. **Unify the scoring layer underneath; keep both surfaces.** Intent docs frame intents as the successor to lenses, explaining the vocabulary overlap. | High |
| Q3 | Monorepo root semantics? | **Nearest-marker-wins default + explicit `--root` override + scope symbol search to target subpath and language.** Correction to the review: `ProjectManifest::detect` (the ".git overrides Cargo.toml" function) has **zero callers — dead code**. The CLI uses `find_project_root`, already nearest-marker. The real zoom bug is `SymbolResolver::find_symbol` walking the entire tree unscoped by language/path — that is how a Rust lookup resolved into deprecated Python. MCP's explicit-root + containment model (post-C1) is the template. | High (mechanism) / Medium (product intent) |
| Q4 | What should "Dark Matter" mean? | **Both: rename the current metric to "Unparsed regions" AND ship real complexity.** Cheaper than it looks: all three main adapters already extract control-flow into IR `Block`/`ControlFlow` types, but `extract_body` has zero non-test callers — the missing wire is attaching `Block` to `Declaration` in the production parse path and having census count decision points. Caveat: complexity only covers the ~3 adapter languages; label the rest honestly rather than showing 0. | High |
| Q5 | Hand-rolled MCP server or rmcp? | **Keep hand-rolled, harden it; delete/quarantine the rmcp prototype.** Hand-rolled: synchronous, 52 tests, best-tested module in repo. rmcp prototype: 0 tests, tokio dependency, incompatible contract, undocumented. Revisit only if the MCP spec evolves fast on schema/version/streaming. | High |
| Q6 | Primary user: humans or agents? | **Agents over MCP are primary; CLI is a preview/debug surface.** No new metaphor work until existing metrics are honest. Recent commit history is already almost entirely MCP/security/test work. (Market bet, partly non-code-derived.) | Medium |
| Q7 | `experiments/` / `classic/` policy? | **Exclude `experiments/`, `classic/`, `.llm_archive/`, `benches/` from default analysis** (walker AND explorer). Scope excludes to repo root, not any-depth, to avoid hiding legitimately-named user dirs. The LSP PoC conclusion is banked under Q1. | High |
| Q8 | `pm_encoder` compatibility? | **No external dependents — delete aggressively.** One release of deprecation warning for `.pm_encoder_config.json`, then drop. Fix `ZOOM_AFFORDANCE` strings immediately regardless. | High |
| Q9 | How real is the learning loop? | **Write-only telemetry today, confirmed by exhaustive trace.** `blend_priority` is called only when `context_store` is `Some`, and production `LensManager` is never constructed with a store. The read-back path is ~one line from working (`with_store(ContextStore::load_from_file(...))`). **Decision: wire it (it's the moat)** — but add decay/exploration logic so early low scores don't permanently bury files. | High |
| Q10 | Token estimation accuracy? | **Calibrate `len/4` per format now** (this is the real root cause of the I1 100%-vs-118% contradiction — flat per-file overhead constant undercounts real serialization overhead). **Feature-gate a real tokenizer later; let MCP clients supply counts** (MCP-first per Q6). Do not bundle one model's tokenizer and call it "exact" — an honest ±10% heuristic beats a misleading exact-for-the-wrong-model count. | High (calibration) / Medium (tokenizer choice) |

## Cross-cutting conclusion

Q1, Q4, and Q9 are the same disease: `core/syntax/adapter.rs`, `extract_body`'s control-flow extraction, and `set_store` are each **complete, tested, and callerless**. The highest-leverage change is process, not code: **a feature is "done" only when a production entry point calls it and an end-to-end test exercises it** — and `--help` text is generated from the same registry that validates input. Three of the ten questions dissolve the moment that rule is enforced.
